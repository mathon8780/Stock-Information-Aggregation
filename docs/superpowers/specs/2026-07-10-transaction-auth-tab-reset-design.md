# Transaction 认证页签归位设计

## 目标

Transaction 的认证区域在普通账户登录、创建后自动登录及任何退出路径后，下一次显示认证区域时始终选中“登录账户”页签。

## 行为范围

- 普通账户从“登录账户”登录成功时，将认证页签状态归位为 `login`。
- 普通账户从“创建账户”创建并自动登录成功时，将认证页签状态归位为 `login`。
- 普通账户主动退出、登录令牌失效或本地会话被清除时，将认证页签状态归位为 `login`。
- 管理员登录会清除普通账户会话；管理员退出并重新显示统一认证入口时，也显示“登录账户”页签。
- 登录后的账户工作台、管理员页面、登录接口和账户数据均不改变。

## 实现方式

沿用 `PaperTrading` 现有的 `authTab` 状态，在所有会话边界调用同一个页签归位方法。会话边界包括普通登录成功、创建并登录成功、普通会话清除和管理员会话清除。这样无需引入新的认证状态机，也不会改变现有路由。

## 验证

- 回归测试覆盖从 `create` 和 `admin` 页签发生会话边界后均返回 `login`。
- TypeScript 编译及前端生产构建通过。
- 浏览器检查创建后退出、普通登录后退出、管理员退出三个流程，认证区均落在“登录账户”。
